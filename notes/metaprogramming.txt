Metaprogramming, it's in the name!

Bringing metaprogramming to my c codebase has always been something that has facinated me for the longest time. To my knowledge, MSVC does not include any "compiler as library" features,
This leaves us with Clang and TCC. While I would much rather prefer to use TCC > Clang, for building *performant* applications it'll just be a deal breaker. I'll revisit this though, as TCC is probably
my favorite compiler. Anyway, with clang we basically have two options: (https://clang.llvm.org/docs/index.html) Libclang, and Compiler plugins. The documentation is wildly unhelpful (per usual) 
so I'm gonna have to look in to each and try to figure out which one supports what I want to do. I have a few goals for this project and the codebase that are heavily inspired by Jai's metaprogramming:
1. Evolve from bash to a scripting language (like lua)
2. Allow each project to contain their own metaprogram
3. Facilitate traversal of the AST
4. Facilitate modification of the AST
5. Reflection?

It would be nice also if I could figure out an easy way to do this as well. If compiler plugins require a ton of modern c++ I would rather try and write it in lua, but I'm not sure yet
how all the pieces would fit together.
